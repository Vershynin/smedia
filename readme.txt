-=Структура папок =-

app - папка разработки

app/css - компиляция css 
app/fonts - подключение шрифтов
app/img - картинки
app/js - js
app/libs - плагины
app/sass - папка с sass файлами
header.sass- стили первого экрана
app/sass/_base.sass - базовые настройки
_libs - подключение стилей плагинов
_media - адаптивность пишем здесь
_vars - переменные sass
fonts - подключение шрифтов
main - главные стили


dist - папка готовой верстки
node_modules - папка с пакетами для gulp
gulpfile.js - папка с настройками gulp
gitigonre - если работаете с Git , игнорирует изменения файлов прописаных там.


-=Запуск Gulp=-

1.Shift + right click (http://joxi.ru/brRp17LteVYM21) в папке с проектом именно по такому пути.Т.е. открыть консоль  в папке где лежит файл gulpfile.js

2. В конcоль npm i (или npm install) (Важно: в x32 и x64 скачиваются разные модули, учтите это иначе может возникнуть ошибка)

3. После установки всех пакетов пишем в консоле gulp

-=Основные требования=-

1.Просьба сетку bootstrap не крыть другими классами. т.е(<div class="col-md-6 my_class"> - не желательно)
2.Инлайновые блоки можно выравнивать через flex
3.В хедере подключена сетка bootstrap.Если нужен полный подключайте через gulp

-=gulpfile.js=-

В основном здесь вы будете подключать js различных плагинов, которые в последствии будет компилиться в libs.min.js
Так же весь sass которые вы пишете , а так же css плагинов которые вы подключаете будет компилиться в файл main.min.css 

-=libs.sass=-

Здесь подключаете css плагинов , но будьте так же внимательны.Если подключаете с расширением css , то в конце его не указываете т.е. (Вместо example.css => example) Иначе gulp будет ругаться.Так же gulp ругается на неправильный синтаксис sass
